## Release note corrections

* The following correction applies to the [8.0.2.0 iFix 2 release notes](https://www.ibm.com/support/pages/node/6614713).

  **PO09775, WorkItem:278381 - The date picker is unnecessarily opened when certain modal dialogs open**

  The issue resolution should read as follows:

    **Resolution:**

    When a user opens a modal dialog that has the date picker form control as the first field, the focus now goes to the modal dialog body instead of the first input field.

* The following correction applies to the [8.0.2.0 release notes](https://www.ibm.com/support/pages/node/6583491).

  **WorkItem:276468 - Modals in Social Program Management application modules are now sized based on their content**

  The link to the UI Upgrade helper should be: [https://merative.github.io/spm-ui-upgrade-helper/](https://merative.github.io/spm-ui-upgrade-helper/)

* The following correction applies to the [8.0.1.0 release notes](https://www.ibm.com/support/pages/node/6516652).

  **WorkItem:272250 - Some modals are not sized based on the contents they are displaying**

  The links to the UI Upgrade helper should be: [https://merative.github.io/spm-ui-upgrade-helper/](https://merative.github.io/spm-ui-upgrade-helper/) and [https://github.com/merative/spm-ui-upgrade-helper](https://github.com/merative/spm-ui-upgrade-helper)
  
* The following correction applies to the [8.0.0 release notes](https://www.ibm.com/support/pages/node/6471277).

  **WorkItem:268106 - Cúram timer-based JMX statistics updated to use Java Timers in IBM WebSphere® Liberty deployments**

  The link to the Social Program Management on Kubernetes runbook should be: [https://merative.github.io/spm-kubernetes/monitoring/jmx-statistics-performance-monitoring/](https://merative.github.io/spm-kubernetes/monitoring/jmx-statistics-performance-monitoring/)
